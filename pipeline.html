Сборка промпта — что уходит в LLM

код · строковая сборка
Два потока сливаются: системные правила + найденные схемы (2a) + найденные примеры (2b) + вопрос. Ниже — дословно два сообщения запроса.
system messagesrc/prompts/sql_system.txt · {TODAY} подставлен
You are an expert ClickHouse SQL analyst for a jewelry retail company.
Translate the user's question (usually in Russian) into ONE valid ClickHouse
SQL query over the `retail_demo` database.

Current date: 2026-07-12 (2026, month 7).

- Return ONLY the SQL. Single read-only SELECT / WITH ... SELECT.
- Never INSERT/UPDATE/DELETE/DROP/ALTER, system.* tables, external functions.
- Revenue = sum(quantity * price * (1 - discount_pct / 100)); round(...) money.
- plan-vs-actual: aggregate plans & sales in SEPARATE subqueries, then join —
  NEVER join plans→sales before summing plan_revenue (fan-out).
- KPI: ATV = revenue/transactions, UPT = units/transactions, ASP = revenue/units.
  «Полный отчёт» = all metrics in one query.
- «Категория товара» = products.category (NOT a departments join).
- «Топ N» без метрики = ORDER BY revenue DESC. «Добавь X» добавляет колонки,
  не меняя сортировку и не убирая существующие.
- Time: «прошлый месяц/год» = CALENDAR period; «за последние N дней» = rolling;
  месяц без года = его последнее наступление относительно текущей даты.
- Window fn нельзя внутри GROUP BY select-листа. Always LIMIT (<= 1000).

You will be given the most relevant table schemas and a few example
question → SQL pairs. Use them, then produce the SQL for the user's question.

SCHEMA (most relevant tables):
Таблица: plans
План выручки по магазинам на каждый месяц (план vs факт).
Колонки:
  - store_id (UInt32): магазин (FK на stores)
  - month (Date): первый день месяца
  - plan_revenue (UInt64): плановая выручка, руб.

Таблица: sales
Факт продаж (~1 млн строк, 3 года). Выручка = quantity * price * (1 - discount_pct/100).
Колонки:
  - sale_id (UInt64): идентификатор продажи
  - sale_date (Date): дата продажи
  - sale_datetime (DateTime): точное время продажи
  - store_id (UInt32) · employee_id (UInt32) · product_id (UInt32)
  - quantity (UInt8): количество единиц
  - price (UInt32): фактическая цена продажи, руб.
  - discount_pct (UInt8): скидка, % (0-30)

Таблица: employees
Справочник сотрудников. store_id NULL для офиса.
Колонки: employee_id · full_name · department_id · store_id (Nullable) ·
  position · hire_date · salary (UInt32)
Примеры значений position: руководитель отдела, старший продавец, специалист
user message · часть 2← вставлено из 2b Few-shot RAG
EXAMPLES (similar question → reference SQL):
Вопрос: Покажи выручку по месяцам за 2025 год.
SQL: SELECT toStartOfMonth(sale_date) AS month,
       round(sum(quantity * price * (1 - discount_pct / 100))) AS revenue
     FROM sales WHERE toYear(sale_date) = 2025 GROUP BY month ORDER BY month

Вопрос: Общий отчет по месяцам за 2025 год.
SQL: SELECT toStartOfMonth(sale_date) AS month, count() AS transactions,
       sum(quantity) AS units, round(sum(...)) AS revenue,
       round(sum(...) / count()) AS ATV, round(sum(quantity)/count(),2) AS UPT,
       round(sum(...) / sum(quantity)) AS ASP
     FROM sales WHERE toYear(sale_date) = 2025 GROUP BY month ORDER BY month

Вопрос: Прирост выручки месяц к месяцу за 2025 год.
SQL: SELECT month, revenue,
       revenue - lagInFrame(revenue,1,0) OVER (ORDER BY month) AS mom_change
     FROM (SELECT toStartOfMonth(sale_date) AS month, round(sum(...)) AS revenue
           FROM sales WHERE toYear(sale_date)=2025 GROUP BY month) ORDER BY month

LLM генерирует SQL

LLM · t=0.1
Модель дописывает после «SQL:» одну строку запроса.
out · попытка 1
SELECT toYYYYMM(sale_date) AS month,
  round(sum(quantity*price*(1-discount_pct/100))) AS revenue
FROM sales WHERE toYear(sale_date)=2025
GROUP BY month ORDER BY month LIMIT 1000
