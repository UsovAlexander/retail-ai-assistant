В сети терабайты продаж, но чтобы ответить на простой вопрос, нужен аналитик, который напишет SQL. Ассистент убирает этого посредника — безопасно и локально. The chain has terabytes of sales, but answering a simple question needs an analyst to write SQL. The assistant removes that middleman — safely and locally.

1

Аналитик — бутылочное горлышкоThe analyst is a bottleneck

Каждый вопрос «сколько, где, кто лучше» идёт через человека с SQL. Очередь растёт, решения ждут. Every "how much, where, who's best" runs through a person writing SQL. The queue grows, decisions wait.

SQL

Барьер для сотрудникаA barrier for staff

Директор магазина не пишет ClickHouse-запросы с оконными функциями и план-vs-факт. Ему нужен ответ. A store manager doesn't write ClickHouse queries with window functions and plan-vs-actual. They just need an answer.

on-prem

Формулировки отличаются от few-shot — меряем генерализацию. Результат сверяется по денотации (множество строк, инвариантно к порядку), а не по тексту SQL. The wordings differ from the few-shot set — we measure generalization. Results are compared by denotation (the set of rows, order-invariant), not by SQL text.

Execution vs Result

Две метрикиTwo metrics

Execution — SQL исполнился. Result — результат совпал с эталоном. Строгая: лишняя колонка = провал. Execution — the SQL ran. Result — the output matched the reference. Strict: an extra column = a miss.

Retry работаетRetry works

≤3 попытки≤3 attempts

Ошибка ClickHouse уходит обратно в модель текстом. Внешней 70B ретраи не нужны — 1.00 попытки. A ClickHouse error goes back to the model as text. The external 70B needs no retries — 1.00 attempts.

АвтофолбэкAuto-fallback

auto = Groq → local

В проде: сначала внешняя, при лимите/сбое — мгновенно локальная. Проверено на живом 429 от Groq. In prod: external first, and on a rate limit or failure — instantly local. Verified against a live 429 from Groq.

ИнженерияEngineering

Принципы и стекPrinciples & stack

Без LangChain — весь конвейер написан явно: маршрутизация, RAG, валидация, ретраи, оценка. Понимание механики важнее вызова фреймворка. No LangChain — the whole pipeline is written explicitly: routing, RAG, validation, retries, evaluation. Understanding the mechanics beats calling a framework.

Принцип 01Principle 01

Инварианты — в кодеInvariants in code

Валидатор, row policy, проверенные факты. Что обязано быть верным — вынесено из LLM. Validator, row policy, verified facts. Whatever must be true is taken out of the LLM.

Принцип 02Principle 02

LLM в сэндвичеLLM in a sandwich

Каждый вероятностный шаг обёрнут детерминированными проверками до и после. Every probabilistic step is wrapped in deterministic checks before and after.

Принцип 03Principle 03

Ядро без интерфейсовCore without interfaces

Один core.ask() с файлами-артефактами. Streamlit и Telegram — тонкие обёртки. One core.ask() with file artifacts. Streamlit and Telegram are thin wrappers.
